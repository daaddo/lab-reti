Introduzione

Requisiti dei servizi

  • Posta elettronica: 1 server
  • Web: 1 server
  • DNS: 2 server
  • Backup: 1 server

Indirizzamento e subnetting

Gli utenti sono complessivamente 310 distribuiti in 5 edifici. Per la rete interna si adotta un indirizzamento privato con subnetting per edificio.

  • Rete privata: 192.168.X.X
  • Subnet mask: 255.255.255.0 (/24)
  • Una sottorete per edificio (fino a 254 host utilizzabili per sede); sottoreti aggiuntive saranno attivabili in seguito.

Topologia e routing

  • Topologia a stella tra A, B, C, D con tratto a bus D–E.
  • Accesso alla rete esterna gestito nell’edificio B.
  • Ogni edificio è dotato di un router; collegamenti inter‑sede in fibra ottica.
  • Protocollo di routing interno: RIP, adeguato alle dimensioni dell’infrastruttura.

Sicurezza e connettività

  • Firewall per gestire l’accesso ai servizi pubblici (web e posta) ospitati in B.
  • Access Point Wi‑Fi nell’edificio A per la copertura wireless.
  • Server di backup in C (elevata interconnessione, carico moderato, assenza Wi‑Fi).

Configurazione Dispositivi

Network Configuration

Linux/Bash 
# Configurazione interfaccia di rete
sudo ip addr add 192.168.1.1/24 dev eth0
sudo ip link set eth0 up

# Route di default
sudo ip route add default via 203.0.113.1

# Configurazione iptables firewall
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT
sudo iptables -A INPUT -j DROP

# Salvataggio configurazione
sudo iptables-save > /etc/iptables/rules.v4

Note di Configurazione:

  • Configurazione IP statico con iproute2
  • Gateway di default verso ISP
  • Regole firewall con iptables
  • Salvataggio persistente delle regole
